# reStructuredText + Sphinx 写作技能

本技能旨在帮助您使用reStructuredText格式编写Sphinx文档的生物信息学教程内容。写作风格强调清晰、平实的教学语言，适合教育材料。

## 写作风格指南

### 语言和语气
编写教程内容时，请全程使用平实、直接的教学语言。目标是让复杂的生物信息学概念对首次学习这些主题的学生和研究人员变得易于理解。在解释中避免使用项目符号或编号列表。相反，应该用长段落写作，让内容从一个概念自然流畅地过渡到下一个概念。这种方法帮助读者跟随思想的逻辑递进，避免因不断在列表项之间切换而产生的思维中断。

保持语言简单易懂，但不要试图在描述中过于创意或生动。重点应放在清晰和准确上，而非娱乐价值。避免在术语或概念周围使用引号，除非它们是实际代码或命令中需要引用的部分。这能保持文本整洁并减少视觉混乱。

### 代码示例
每当解释可以用代码演示的概念时，请在教程中直接嵌入可执行的代码块。使用带有适当语言说明符的reStructuredText代码块语法。例如，在解释R编程概念时，使用R代码块。在解释命令行操作时，使用bash或shell代码块。这些代码示例应尽可能完整且可运行，以便读者可以复制并执行它们以查看结果。

**重要：代码块位置要求**
代码块必须紧跟在相关文字解释之后，而不是集中在章节末尾。每个代码块应该作为对紧邻的前文内容的补充和演示。这种一一配对的方式确保读者能够立即看到理论如何转化为实践，增强学习效果。代码块与文字之间应该用空行分隔，但不要有其他内容插入。

### 写作策略（两步法）
为了提高写作效率和内容质量，采用两步法进行教程编写：

**第一步：编写文本内容**
首先编写解释性的文字内容，使用平实的教学语言详细阐述概念和原理。这一步专注于文字叙述，确保逻辑清晰、内容完整。文本应该能够独立存在，即使没有代码示例也能让读者理解基本概念。在这一步中，不要插入代码块，只专注于文字描述。使用长段落进行叙述，避免使用分点或编号列表。

**第二步：添加代码示例**
在文本内容完成后，回过头来为每个适合的概念添加可执行的代码块。代码块应该紧跟在相关文字解释之后，作为对文字内容的补充和演示。每个代码块应该包含详细的注释，解释代码的每一部分在做什么。注释应该使用中文，与文本内容保持一致的语言风格。

这种两步法的好处是：
1. 第一步让作者专注于概念解释和逻辑组织，避免在写作过程中频繁切换思维模式
2. 第二步专注于代码实现和示例演示，确保代码块位置正确（紧跟相关文字）
3. 提高了写作效率和内容质量
4. 确保代码块与文字讲解一一配对，而不是集中在章节末尾

### 内容结构
使用清晰的章节标题和小节来组织教程。使用带有适当下划线字符的reStructuredText标题语法。从介绍开始，解释教程将涵盖的内容及其重要性。然后按逻辑顺序推进材料，从基础概念构建到更高级的主题。每个部分应自然地过渡到下一个，通过过渡帮助读者理解概念之间的联系。

### 交互元素
随时对用户提供的内容提出问题或建议改进。如果您注意到解释中的空白或可以从额外细节中受益的区域，可以添加补充信息。您也可以主动指出潜在的混淆点并提供澄清。这种交互方法有助于确保教程内容尽可能有帮助和完整。

## 代码块格式规范

### 基本语法
使用reStructuredText的code-block指令创建代码块：

```rst
.. code-block:: python

    # 代码内容
    print("Hello World")
```

### 代码块位置
代码块必须紧跟在相关文字说明之后，中间只留一个空行：

```rst
这是一段关于Python变量定义的文字说明。变量是存储数据的容器，在生物信息学中经常用来存储基因名称、序列数据等信息。

.. code-block:: python

    # 定义基因名称变量
    gene_name = "TP53"
    print(f"正在分析基因: {gene_name}")

这是另一段文字说明，继续讲解下一个概念。
```

### 错误示例（避免）
不要将所有代码块集中在章节末尾：

```rst
这是第一段说明。

这是第二段说明。

这是第三段说明。

.. code-block:: python

    # 第一个代码块（错误：应该放在第一段说明之后）
    print("代码块1")

.. code-block:: python

    # 第二个代码块（错误：应该放在第二段说明之后）
    print("代码块2")
```

### 正确示例
代码块应该紧跟在相关文字之后：

```rst
这是第一段说明。

.. code-block:: python

    # 第一个代码块（正确：紧跟第一段说明）
    print("代码块1")

这是第二段说明。

.. code-block:: python

    # 第二个代码块（正确：紧跟第二段说明）
    print("代码块2")

这是第三段说明。
```

理解生物信息学中的文件路径

在处理生物信息学数据时，您经常需要指定文件在计算机上的位置。这些位置称为文件路径，它们告诉操作系统确切在哪里找到特定文件。文件路径可以是绝对路径或相对路径，理解它们之间的差异对于有效组织生物信息学项目至关重要。

绝对路径指定从文件系统根目录开始的文件的完整位置。在Windows系统上，这通常以驱动器字母（如C:）开头，后跟分隔每个目录级别的反斜杠。在基于Unix的系统（如Linux和macOS）上，绝对路径以正斜杠开头。当您需要引用位于当前工作目录之外的文件，或者想要完全明确地指定文件位置时，绝对路径很有用。

相对路径则指定相对于当前工作目录的文件位置。这些路径更短且更具可移植性，意味着只要保持相对目录结构，它们就可以在不同的计算机上工作。在生物信息学项目中，通常首选相对路径，因为它们使您的分析脚本更具可重复性，并且更容易与协作者共享。

让我们看一些在R中处理文件路径的实际示例：

    .. code-block:: r

        # 使用绝对路径读取文件
        data <- read.csv("C:/Users/username/data/expression.csv")

        # 使用相对路径读取文件
        data <- read.csv("data/expression.csv")

        # 获取当前工作目录
        getwd()

        # 更改工作目录
        setwd("C:/Users/username/project")

当您在脚本中使用相对路径时，您可以将整个项目目录移动到不同位置或与同事共享，只要内部目录结构保持不变，所有文件引用将继续正确工作。这种可移植性在协作研究项目中特别重要，因为不同的团队成员可能在不同的计算环境中工作。

## 何时使用此技能

在以下情况下调用此技能：
- 以reStructuredText格式创建新的生物信息学教程内容
- 更新或扩展现有教程章节
- 将解释或笔记转换为正确格式的教程文本
- 向教程内容添加代码示例和演示
- 审查和改进现有教程写作风格
- 编写生物信息学相关的技术文档和教程

## 最佳实践

1. 始终考虑您的受众。生物信息学教程通常服务于可能是生物学专家但编程新手的学生和研究人员，反之亦然。编写能够弥合这些不同背景的解释。
2. 测试您的代码示例以确保它们按描述工作。如果您不确定代码示例是否能正确运行，请在教程中注明并建议读者可能需要如何调整它。
3. 提供每个概念为何重要的背景。当读者理解信息如何应用于实际的生物信息学问题时，他们会学得更好。
4. 逐步构建概念。从简单示例开始，随着读者对材料越来越熟悉，逐渐引入复杂性。
5. 预期常见错误并在解释中主动解决它们。这有助于读者避免挫折并建立信心。
6. 在引入新术语时，在教程中广泛使用之前清楚地定义它。
7. 将新概念与先前涵盖的材料联系起来，以加强学习并展示不同主题之间的关系。

8. 确保代码块紧跟在相关文字说明之后，实现理论到实践的无缝过渡。

9. 使用真实的生物信息学场景和数据作为代码示例，而不是抽象的示例。

10. 在代码注释中提供详细的中文解释，帮助读者理解每一行代码的作用。


## 内容编写原则

### 语言风格
- 使用简单易懂的语言
- 避免使用初学者不理解的专业术语
- 如需使用专业术语，必须在相应位置进行解释

### 教学方式
- 从简单到复杂，循序渐进
- 知识点配有可执行的代码示例
- 提供实际应用场景说明

### 代码示例

- 示例要贴近实际应用
- 避免过于抽象的示例

### 实际应用场景融入（重要！）

每个知识点都要说明其实际应用场景，但要自然地融入讲解中，不要使用显式的"意义与使用场景"标题。应该在介绍概念或工具时，用流畅的叙述方式说明它在什么情况下会用到、能解决什么问题。例如：

**不推荐的方式**：
```
BLAST是序列比对工具。

**意义与使用场景**：BLAST用于序列功能注释。
```

**推荐的方式**：
```
BLAST是序列比对工具。当你获得一个新的序列（如克隆的基因、测序结果），需要了解它的功能时，第一步通常就是进行BLAST搜索，找出数据库中与之相似的已知序列，从而推断其可能的功能。
```

## 数学知识编写规范

### 1. 数学公式格式

**行内公式**：使用单个美元符号 `$公式$`
```markdown
样本均值 $\bar{X}$ 的计算公式为...
```

**独立公式**：使用双美元符号 `$$公式$$`
```markdown
$$t = \frac{\bar{X} - \mu_0}{s/\sqrt{n}}$$
```

### 2. 数学公式章节结构

当章节包含数学知识时，按以下结构组织：

```markdown
## X.X 章节标题 {#xx-章节标题}

### 数学基础

**概念名称**：

简要解释数学概念的含义和用途。

**公式**：
$$公式$$

**参数说明**：
- 参数1：含义
- 参数2：含义

**适用条件**：
1. 条件1
2. 条件2

```{r}
# 代码示例
# 注释说明如何用R实现该数学概念
```
```

### 3. 数学公式编写要求

#### 公式必须包含
1. **公式本身**：使用LaTeX语法
2. **参数说明**：解释公式中各符号的含义
3. **适用条件**：说明公式的使用前提
4. **代码实现**：用R代码演示公式的计算过程

### 4. 数学概念解释原则

1. **先直观后严谨**：先用通俗语言解释概念，再给出数学定义
2. **结合实例**：每个数学概念都要配具体数值例子
3. **可视化**：尽可能用图形展示数学概念
4. **代码验证**：用R代码验证数学公式的正确性

### 5. 数学章节示例

```markdown
## 4.3 p值 {#43-p值}

### 数学基础

**p值（p-value）**：

在原假设H₀为真的条件下，观测到当前样本或更极端结果的概率。

**计算公式**：
$$p = P(T \geq t_{obs} | H_0)$$

**参数说明**：
- $T$：检验统计量
- $t_{obs}$：观测到的检验统计量值
- $H_0$：原假设

**解释**：
- p值越小，拒绝H₀的证据越强
- 通常 p < 0.05 时拒绝H₀

**适用条件**：
1. 需要明确的原假设
2. 已知检验统计量的分布

```{r}
# p值计算示例
observed_t <- 2.5
df <- 19

# 双侧检验的p值
p_value <- 2 * pt(-abs(observed_t), df = df)
cat("p值 =", p_value, "\n")
```
