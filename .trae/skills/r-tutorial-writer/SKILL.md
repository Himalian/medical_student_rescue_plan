# R语言教程编写器

这个skill用于编写R语言教程的R Markdown文档，遵循以下规范：

## 文档结构要求

### YAML头部
```yaml
---
title: "教程标题"
author: "小天使格礼"
output: html_document
---
```

### 安装包提示（重要！）

**位置**：在YAML头部之后、setup代码块之前

**格式**：使用 `eval=FALSE` 标记的代码块，包含所有需要的包

```markdown
在Rstudio中安装本节课所有需要的包，运行以下指令：
```{r, eval=FALSE}
install.packages(c(
  "dplyr", "tidyr", "ggplot2",
  "psych", "pastecs", "e1071",
  "boot", "pwr", "reshape2",
  "car", "DescTools", "MASS",
  "vcd", "exact2x2", "coin",
  "multcomp", "emmeans", "afex",
  "ggpubr", "rstatix",
  "corrplot", "ppcor",
  "lm.beta", "broom"
))
```
```

**要求**：
1. 安装包列表必须与setup代码块中的library()调用一致
2. 使用 `eval=FALSE` 防止每次knit时重复安装
3. 包名按功能分组排列，便于阅读
4. 提示文字使用简单明了的语言

### Setup代码块
```r
```{r setup, include=FALSE}

```
```

### 目录格式（可点击跳转）
```markdown
| 章节 | 主题 | 核心内容 |
|:----:|:----:|:--------:|
| [第一章](#第一章章节标题) | 主题描述 | 核心内容 |

**注意**：锚点格式为 `#章节标题`，需要将标题中的空格替换为连字符，特殊字符去掉。
```

## 代码编写规范

### 1. 代码注释要求
- **每个代码块必须有详细注释**
- 注释使用简单易懂的语言，适合R语言初学者
- 关键函数需要说明参数含义
- 复杂逻辑需要分步注释
- 注释使用中文

### 2. 注释风格示例

```r
# 创建数值向量x，包含1到5
x <- c(1, 2, 3, 4, 5)

# 计算向量x的平均值
mean_x <- mean(x)

# 输出结果
mean_x
```

### 3. 代码块格式
- 不要在代码块中手动写输出结果（如 `##>` 格式）
- R在knit时会自动执行代码并显示输出
- 使用 `eval=FALSE` 标记不执行的代码块（如安装包代码）

### 4. 函数显式调用规范（重要！）

**原则：所有函数优先使用显式调用 `包名::函数名()`，避免包冲突**


#### 示例

```r
# 错误：可能与其他包冲突
p + theme(
  plot.margin = margin(10, 10, 10, 10)
)

# 正确：显式指定ggplot2包
p + theme(
  plot.margin = ggplot2::margin(10, 10, 10, 10)
)

# 错误：filter可能被stats::filter覆盖
filter(df, x > 5)

# 正确：显式指定dplyr包
dplyr::filter(df, x > 5)
```

#### 规则
1. **ggplot2相关函数**：`margin()`、`element_text()`、`element_rect()` 等theme相关函数必须使用 `ggplot2::` 前缀
2. **dplyr函数**：`filter()`、`select()`、`mutate()`、`summarize()` 等必须使用 `dplyr::` 前缀
3. **其他可能冲突的函数**：任何在多个包中存在的同名函数，都必须显式指定包名

### 5. 表格格式
- **重要：表格必须使用纯Markdown格式，不要使用R代码块生成表格**
- 表格标题后需要空一行
- 使用标准Markdown表格格式：

```markdown
| 列1 | 列2 | 列3 |
|:---:|:---:|:---:|
| 内容 | 内容 | 内容 |
```

**禁止使用以下方式生成表格**：
```r
# 错误：使用kableExtra生成表格
kableExtra::kable(data, caption = "表格标题") %>%
  kableExtra::kable_styling(bootstrap_options = c("striped", "hover"))

# 错误：使用knitr::kable生成表格
knitr::kable(data)
```

**正确做法**：直接使用Markdown语法编写表格
```markdown
| 学习类型 | 数据特点 | 学习目标 |
|:--------:|:--------:|:--------:|
| 监督学习 | 输入-标签对 | 预测标签 |
| 无监督学习 | 无标签数据 | 发现结构 |
```

### 6. 章节结构
- 使用 `#` 表示章
- 使用 `##` 表示节
- 使用 `###` 表示小节
- **注意**：章节之间不要使用 `---` 分隔符（会导致pandoc YAML解析错误）

## 内容编写原则

### 语言风格
- 使用简单易懂且生动的语言
- 避免使用初学者不理解的专业术语
- 如需使用专业术语，必须在相应位置进行解释

### 教学方式
- 从简单到复杂，循序渐进
- 每个知识点配有可执行的代码示例
- 提供实际应用场景说明
- 使用表格总结关键知识点

### 代码示例
- 所有代码必须可执行（除非标记 `eval=FALSE`）
- 示例要贴近实际应用
- 避免过于抽象的示例

### "意义与使用场景"编写规范（重要！）

每个知识点都要尽可能有“意义与使用场景”这个模块

每个知识点的"意义与使用场景"部分必须遵循以下格式：

#### 1. 结构要求

```markdown
### 意义与使用场景

**核心意义**：一句话概括该方法的核心作用。

**举例**：具体的研究案例描述

**为什么这种数据适合用这种方法？**

详细分析数据特点与方法适用性的匹配关系，解释变量类型、数据结构等。

**研究目的是什么？**

说明研究的具体目标，可分层次阐述。

**可能得到什么样的结果？**

给出假设的分析结果，并详细解释。

**需要注意什么？**

提醒潜在的假设违背、数据问题或方法局限。
```

#### 2. 写作要求

1. **只举一个例子**：不要列举多个领域的例子，专注于一个典型案例进行深入剖析
2. **深入剖析**：必须包含以下四个部分：
   - 为什么这种数据适合用这种方法
   - 研究目的是什么
   - 可能得到什么样的结果
   - 需要注意什么
3. **列表格式**：使用"1. 2. 3. 4....（不是只能有四点，也不是至少有四点，多少点都行）"换行形式，**严禁使用markdown的"- "格式**（会在knit后渲染为圆点）

#### 3. 列表格式示例

**正确格式**（换行列举）：
```markdown
1. 第一项内容。
2. 第二项内容。
3. 第三项内容。
```

**错误格式**（会导致渲染问题）：
```markdown
- 第一项内容
- 第二项内容
- 第三项内容
```

#### 4. 完整示例

```markdown
### 意义与使用场景

**核心意义**：多元线性回归用于研究一个连续型因变量与多个自变量之间的线性关系，是量化多因素影响、进行预测推断的基础工具。

**举例**：研究年龄、BMI、住院天数对医疗费用的影响

**为什么这种数据适合用线性回归？**

首先，因变量"医疗费用"是连续型数值变量（单位：元），取值范围从几百到几十万不等，符合线性回归对因变量的要求。其次，三个自变量——年龄（连续变量）、BMI（连续变量）、住院天数（连续变量）——都是可量化的数值型指标，且理论上与医疗费用存在近似线性的关系。这种"连续因变量+多个连续自变量+近似线性关系"的数据结构，正是多元线性回归的典型适用场景。

**研究目的是什么？**

本研究有三个层次的目的：1. 量化影响程度：回答"年龄每增加1岁，医疗费用平均增加多少元？"这类具体问题；2. 识别关键因素：在控制其他变量后，判断哪个因素对费用的影响最大；3. 预测应用：建立预测模型，根据患者特征预估医疗费用。

**可能得到什么样的结果？**

假设回归结果为：费用 = -5000 + 200×年龄 + 150×BMI + 800×住院天数

1. 截距解释：当年龄=0、BMI=0、住院天数=0时，理论费用为-5000元（截距本身无实际意义，仅是数学外推）。
2. 系数解释：年龄系数200表示，在BMI和住院天数不变的情况下，年龄每增加1岁，医疗费用平均增加200元。
3. 比较重要性：通过标准化系数或t值，可以判断哪个因素的影响更显著。
4. 预测示例：一位60岁、BMI=25、住院10天的患者，预计费用 = -5000 + 200×60 + 150×25 + 800×10 = 11,750元。

**需要注意什么？**

线性回归假设变量间关系是线性的，但实际中可能存在非线性关系。此外，医疗费用通常右偏分布，可能需要对数变换。这些都需要在建模前进行探索性分析。
```

## 常见问题与解决方案

### 1. 逻辑运算符
- `&&` 和 `||` 只能用于单个逻辑值（if条件判断）
- `&` 和 `|` 用于向量化运算
- 新版本R中 `&&` 不能直接用于向量

### 2. 文件路径
- Windows路径使用正斜杠 `/` 或双反斜杠 `\\`
- 不能使用单反斜杠 `\`

### 3. 表格渲染
- 表格前必须有空行
- 分隔符使用 `|---|` 格式，不是中文破折号

### 4. Pandoc YAML解析错误
- **问题**：文件中间的 `---` 被误认为YAML边界
- **解决**：不要在章节之间使用 `---` 分隔符

### 5. 函数参数版本兼容性
- **问题**：某些包的函数参数在不同版本中可能变化
- **解决**：使用基础R函数替代，或检查参数是否正确
- **示例**：`DescTools::Winsorize()` 的 `probs` 参数可能不被支持
  ```r
  # 替代方案：手动实现
  lower_bound <- quantile(data, 0.05)
  upper_bound <- quantile(data, 0.95)
  result <- pmin(pmax(data, lower_bound), upper_bound)
  ```

### 6. dplyr中使用 complete.cases()
- **问题**：`filter(data, complete.cases(.))` 中 `.` 无法正确解析
- **解决**：使用基础R语法
  ```r
  # 错误写法
  filter(data, complete.cases(.))
  
  # 正确写法
  data[complete.cases(data), ]
  # 或使用 drop_na()
  drop_na(data)
  ```

### 7. 断言检查导致knit失败
- **问题**：`assert_that()` 在数据不满足条件时会停止执行
- **解决**：先处理数据，再对清洗后的数据进行断言
  ```r
  # 错误做法：直接对原始数据断言
  assert_that(all(data$value > 0))
  
  # 正确做法：先清洗再断言
  clean_data <- filter(data, value > 0)
  if (nrow(clean_data) < nrow(data)) {
    message("发现异常记录，已处理")
  }
  assert_that(all(clean_data$value > 0))
  ```

### 8. 包版本依赖问题
- **问题**：`namespace 'rlang' x.x.x is already loaded, but >= x.x.y is required`
- **解决**：更新相关包
  ```r
  install.packages("rlang")
  # 或更新所有包
  update.packages(ask = FALSE)
  ```

### 9. 管道符 %>% 找不到函数
- **问题**：`could not find function "%>%"`
- **原因**：管道符 `%>%` 来自 `magrittr` 包，通常通过 `dplyr` 加载
- **解决**：在setup代码块中加载 `dplyr` 包
  ```r
  # 在setup代码块中添加
  library(dplyr)  # 提供 %>% 管道符
  ```

### 10. 必须在setup代码块中加载所有需要的包
- **问题**：代码块中使用某个函数时报错"could not find function"
- **原因**：包未在setup中加载，或只在某个代码块中加载（其他代码块无法使用）
- **解决**：
  1. 检索文件中所有 `library()` 调用
  2. 将所有需要的包统一添加到setup代码块
  3. 删除后续代码块中重复的 `library()` 调用
  ```r
  ```{r setup, include=FALSE}
  knitr::opts_chunk$set(echo = TRUE)
  
  # 加载所有需要的包
  library(ggplot2)
  library(dplyr)
  library(tidyr)
  # ... 其他所有包
  ```
  ```

### 11. 包已弃用不可用
- **问题**：`package 'xxx' is not available for this version of R`
- **原因**：某些包已被CRAN移除或弃用
- **常见弃用包**：`maptools`（地图相关，已被弃用）
- **解决**：
  1. 从安装列表和setup代码块中移除该包
  2. 检查教程中是否有调用该包的代码，如有则删除或替换
  3. 寻找替代包（如地图相关可用 `sf`、`maps`、`rnaturalearth`）
  4. 注意：有时包只是被加载但未实际使用，可直接删除

### 12. 函数名冲突
- **问题**：`no applicable method for 'xxx' applied to an object of class ...`
- **原因**：多个包有同名函数，后加载的包覆盖了前面的函数
- **常见冲突**：
  - `layout()`：基础绘图 `graphics::layout()` vs `plotly::layout()`
  - `filter()`：`dplyr::filter()` vs `stats::filter()`
  - `select()`：`dplyr::select()` vs `MASS::select()`
- **解决**：使用 `包名::函数名()` 显式指定
  ```r
  # 错误：plotly的layout被错误调用
  layout(matrix(c(1,2,3,3), 2, 2))
  
  # 正确：显式指定使用graphics包的layout
  graphics::layout(matrix(c(1,2,3,3), 2, 2))
  ```

### 13. 代码中引用未定义的变量
- **问题**：`object 'xxx' not found`
- **原因**：代码中使用了未定义的变量或对象
- **常见情况**：
  - 示例代码中引用了未创建的图形对象（如p1, p2等）
  - 变量名拼写错误
  - 前面的代码块被标记为`eval=FALSE`但后面引用了其创建的对象
- **解决**：
  1. 确保所有引用的对象都在前面的代码块中定义
  2. 检查变量名拼写
  3. 如果示例需要多个对象，确保都已创建或调整示例

## Knit检查与错误修复流程

### 1. 在Trae中执行Knit

完成Rmd文件编写后，**必须**在Trae中执行knit命令验证文件是否能正确渲染：

```powershell
# 设置环境变量（包含pandoc路径）
$env:PATH = "C:\Program Files\RStudio\resources\app\bin\quarto\bin\tools;" + $env:PATH

# 切换到文件所在目录
Set-Location "文件所在目录的绝对路径"

# 执行knit
& "C:\Program Files\R\R-4.5.3\bin\Rscript.exe" -e "rmarkdown::render('文件名.Rmd', output_format = 'html_document')"
```

**注意事项**：
1. R的版本号需要根据实际安装版本调整（如R-4.5.3）
2. 目录路径必须是绝对路径
3. 文件名需要包含.Rmd扩展名



### 3. Knit检查流程

每次修改Rmd文件后，按以下流程检查：

1. **执行knit命令**：在Trae中运行渲染命令
2. **检查输出**：
   - 成功：显示 `Output created: 文件名.html`
   - 失败：记录错误信息和代码块位置
3. **定位错误**：根据错误信息找到对应的代码块（如 `[unnamed-chunk-13]`）
4. **修复错误**：根据错误类型应用对应的解决方案
5. **重新knit**：重复直到成功

### 4. 错误定位技巧

错误信息通常包含代码块编号，如：
```
Quitting from R语言与机器学习（下）.Rmd:558-572 [unnamed-chunk-13]
```

- `558-572`：代码块的起始和结束行号
- `unnamed-chunk-13`：代码块名称（未命名时自动生成）

使用Read工具读取对应行号的代码进行分析和修复。

### 目录格式要求
目录应按章节分组，使用表格形式，每行包含：
- 小节编号
- 内容/图形类型描述
- 可点击跳转的链接

示例：
```markdown
### 第一章：章节名称

| 小节 | 内容 | 链接 |
|:----:|:----:|:----:|
| 1.1 | 内容描述 | [描述](#11-标题) |
| 1.2 | 内容描述 | [描述](#12-标题) |
```

### 标题锚点格式
每个章节标题必须添加锚点ID，格式为：
```markdown
## 1.1 标题名称 {#11-标题名称}
```

锚点ID规则：
- 使用小写字母
- 中文保留
- 空格替换为连字符`-`
- 特殊字符（如括号、冒号）替换为连字符或删除

## 使用示例

当用户要求编写R语言教程内容时，按照以上规范：
1. 创建或更新指定的 .Rmd 文件
2. 添加详细注释
3. 确保代码可执行
4. 使用简单易懂的语言解释概念
5. 在YAML之后、setup之前添加安装包提示代码块
6. 在setup中统一加载所有需要的包（与安装包列表一致）
7. 使用可点击跳转的目录格式
8. 包含数学知识时，遵循数学公式编写规范：
   - 使用LaTeX语法编写公式（`$...$` 或 `$$...$$`）
   - 提供参数说明和适用条件
   - 用R代码验证数学公式
   - 先直观解释，后严谨定义
